# v0.7.5 — enforced pipeline tracking

Fixes the recurring bug where the desktop pipeline **freezes mid-run** — the card sits on one step (e.g. EXECUTE) while the terminal keeps working, then jumps straight to Done at the end. The root cause was architectural: pipeline tracking was an advisory *"report"* contract (each `/tiki` step had to remember to run the state shim, with nothing enforcing it). This release flips the write side to a *"reconcile"* contract — the system recomputes pipeline state from the artifacts the work actually produced, so a dropped transition self-heals.

## Fixed
- **Pipeline freezes mid-run / status desync** ([#244](https://github.com/Eric-Ness/Tiki-V2/issues/244)) — a deep-dive confirmed the prior fixes (worktree resolution #211, command drift #230, watcher #224) were sound; the live cause was that intermediate REVIEW/PLAN/AUDIT/EXECUTE transitions were silently dropped while SHIP always ran, producing the freeze-then-jump symptom. Addressed at the architecture level (below), with a runtime test that fails if tracking ever stops advancing.
- **`/tiki:audit` could not record its own step** ([#247](https://github.com/Eric-Ness/Tiki-V2/issues/247)) — `audit.md` was missing `Bash` in its tool list, so a standalone audit couldn't run its transition. Fixed.

## Added
- **State reconciler + enforcement hook** ([#245](https://github.com/Eric-Ness/Tiki-V2/issues/245)) — `reconcile-state.mjs` derives each active issue's true `pipelineStep`/`status`/`phase` from on-disk artifacts (plan file + history), wired as Claude Code `Stop` + `SubagentStop` hooks so it runs every turn. It is activeWork-scoped (never resurrects shipped issues), advance-only, skips `failed`/`paused`/`completed`, derives completion from history (no per-turn GitHub calls), and can never block a turn.
- **AUDIT artifact** — plans now carry `audited`/`auditedAt` (schema + Rust + TS), written by `/tiki:audit` on PASS, so the reconciler can distinguish AUDIT from PLAN.
- **Stale-tracking anomaly** ([#246](https://github.com/Eric-Ness/Tiki-V2/issues/246)) — the detail badge now surfaces a "Pipeline stalled" warning when an in-flight item stops advancing, so a genuinely stuck run is visible rather than appearing stuck silently.
- **Runtime reconciliation tests** ([#248](https://github.com/Eric-Ness/Tiki-V2/issues/248)) — 14 fixture tests including a drop-resilience proof (correct EXECUTE state with zero imperative transitions) and guards for two regressions (a `failed` item is never overridden; a stale all-complete plan is never resurrected).

## Changed
- **Unconditional transitions** ([#247](https://github.com/Eric-Ness/Tiki-V2/issues/247)) — removed the conditional "the skill will emit it, so the parent won't" dispatch trap from `yolo.md`; every step now emits its transition unconditionally (double-emit is a safe no-op).
- **Stable file-sync subscription** ([#249](https://github.com/Eric-Ness/Tiki-V2/issues/249)) — the desktop's `tiki-file-changed` listener now keys on the project *path* instead of the project object, so it no longer tears down and re-subscribes on every framework-version stamp (a window where state writes could be missed).

## Known follow-up
- The reconciler enforcement currently applies in the Tiki repo itself; distributing the scripts + hook to other installed projects is tracked in [#251](https://github.com/Eric-Ness/Tiki-V2/issues/251).
