# v0.2.18

## Features
- **In-app framework update with version stamping** ([#141](https://github.com/Eric-Ness/Tiki-V2/issues/141)) — Framework command files (`.claude/commands/tiki/*.md`) are now embedded in the desktop binary via `include_dir`. When a project's installed framework is older than the running app, a warning toast appears on project switch and an "Update" badge on the project card runs `install_framework` in one click. `install.js` and `install_framework` both stamp `<project>/.tiki/.framework-version` so detection is reliable. `/tiki:version` now reads the stamp file as the authoritative source.
- **Default panel sizes adjusted to 15/65/20** ([#139](https://github.com/Eric-Ness/Tiki-V2/issues/139)) — Detail panel grows from 15% to 20% to fit issue bodies, plan content, and ResearchDetail markdown views; center pane shrinks from 70% to 65%. Existing users keep their tuned layouts (persist version intentionally not bumped); new installs and "Reset Layout" pick up the new defaults.

## Bug Fixes
- **Shipped releases no longer appear in sidebar** ([#142](https://github.com/Eric-Ness/Tiki-V2/issues/142)) — `load_tiki_releases` previously scanned both `.tiki/releases/` and `.tiki/releases/archive/`, leaking shipped releases into the active list. Removed the archive scan; archived releases stay archived. Historical view continues to render from `state.json`'s `history.recentReleases`.
- **In-app update install path is now reachable** ([#140](https://github.com/Eric-Ness/Tiki-V2/issues/140)) — `tauri_plugin_dialog` was registered in Rust but `dialog:default` was missing from the capability file, so `ask()` was rejected silently at the IPC bridge. Added the permission and replaced the silent `console.error` catch with a user-facing error toast. Also added "Downloading update v{version}... {percent}%" and "Installing update..." progress toasts (powered by a new `updateToast` action on the toast store) so the multi-second download/install path is no longer feedback-free.
