# v0.4.1

Seven small enhancements from the `docs/ENHANCEMENT-IDEAS.md` backlog, bundled as a quick-wins follow-up to v0.4.0.

## Features

- **Click an Active Work card to open it in the detail panel** (E5) — `WorkProgressCard` now opens the issue in the detail panel on click, with keyboard support (Enter / Space) and a `cursor: pointer` affordance. Internal action buttons (Pause / Reset / Remove) stop propagation so they don't double-fire.

- **Failed-aware kanban columns** (E9) — when any card in a column has `failed` work status, the column header gains a red border, a warning glyph, and a bold red count. Surfaces failure signal at a glance without scrolling through cards.

- **Command palette inherits the selected-issue context** (E13) — when an issue is open in the detail panel, the bare `tiki:get / plan / execute / ship / yolo` actions are joined by `Run X on #N (current issue)` variants with the number pre-filled. Removes the manual issue-number lookup step.

## Reliability

- **`atomic_write` now `fsync`s the tmp file before rename** (E18) — closes a power-loss durability window where the rename could commit before the data (revert-on-crash). Affects `state.json`, plan files, and release files — every atomic write the backend performs. All 10 `fs_utils` unit tests pass with the new code path.

## Documentation

- **CLAUDE.md updates** (E37) — documents the `state.mjs` shim (the bash-callable mirror of the Rust `state_transition` IPC), warns that `pnpm build` uses `tsc -b` and is stricter than `pnpm typecheck`'s `tsc --noEmit`, and captures the Windows pnpm reparse-point env gotcha for future agents. Adds links to `ENHANCEMENT-IDEAS.md` and `CHANGELOG.md`.

- **`CHANGELOG.md` at repo root** (E40) — human-readable release history loosely following Keep a Changelog. Detailed per-issue notes still live in `.tiki/releases/vX.Y.Z-changelog.md` and on the GitHub Releases page.

## CI

- **`release.yml` composes a real release body from `.tiki/releases/v*-changelog.md`** (E41) — replaces the static "See the assets below…" stub. Falls back gracefully if the per-version changelog file is missing. This release is the first to benefit from it.

## Notes

- **E8** (kanban column count badge) was found already implemented at `KanbanColumn.tsx:42–50` during the exploration pass. Will be marked as `_already shipped_` in `ENHANCEMENT-IDEAS.md` in a follow-up.
- Local pnpm env on the development machine is in a partially-broken state from earlier Windows reparse-point troubleshooting; final TypeScript verification will land via CI. Rust-side changes (the critical `fs_utils::atomic_write` durability fix) were verified locally — all 10 unit tests pass.
