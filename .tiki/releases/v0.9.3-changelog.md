# v0.9.3

Two structural epics that retire the framework's longest-standing reliability bug classes: the pipeline now self-heals from dropped state transitions end-to-end, the state shim has no unvalidated escape hatches, and the type system fails CI on drift.

## Bug Fixes
- **Plugin-only installs now track state** (#268) — command bodies referenced a project-relative `state.mjs` path that didn't exist on a plugin-only install, so every transition threw and the kanban/Active Work never populated. A plugin `SessionStart` hook now bootstraps the scripts into the project; `tiki_doctor` reports unresolved script paths; a behavioral plugin-layout test guards it. (Plugin users: restart the session once after updating.)
- **Skipped phases render correctly** (#274) — the Rust `PhaseProgressStatus` was missing `Skipped` and silently coerced it to `Completed`; a skipped current-phase now round-trips faithfully.

## Reliability — self-healing pipeline (Epic: distribution + self-healing)
- **Reconciler bootstraps missing entries** (#270) — a dropped or failed entry-creating step (e.g. GET) is now self-healing: the reconciler creates the activeWork entry from a recent plan file, guarded against resurrecting long-shipped issues.
- **Reconciler derives SHIP + reconciles releases** (#271) — completion is now also derived from an archived plan + a closed GitHub issue (not just history membership), and `release:*` entries reconcile too; the `gh` fetcher is only consulted for ship-shaped entries so normal passes stay fast.
- **Intent journal** (#272) — every command appends one `.tiki/journal.ndjson` line as its first action; the reconciler reconstructs pipeline position from journal + artifacts, so imperative transitions become an optimization rather than a correctness requirement. This structurally ends the "pipeline froze mid-run" class.

## Reliability — single-source state (Epic: kill the mirrors)
- **Schema-authoritative types** (#274) — the JSON schemas are now the single source for enums/shapes, enforced by schema↔TS and schema↔Rust parity tests that fail CI on drift.
- **Complete shim mutation surface** (#275) — every previously-direct JSON write now has a validated `state.mjs`/`plan.mjs` subcommand (`parallel`, `heal-attempt`, `enrich`, `release-wave`, `phase`, `verify-criteria`, `audited`); all "direct JSON write acknowledged" escape hatches are deleted and guarded.
- **Release status single-source** (#276) — a release's directory location is the sole truth for completion; a new Diagnostics "Normalize archived releases" action and a release-readiness warning clean up and catch stale-status residue (the v0.9.0 footgun).
