# v0.5.4 — Terminal polish (continued)

Five-issue follow-up to v0.5.3's paste fix. Most of these were originally queued for v0.5.3 but split out after build verification was blocked locally. This release closes the bundle (minus the `@xterm/xterm` package migration, which moved to v0.5.5 because it requires a `pnpm-lock.yaml` regeneration that can't be done from this Windows machine right now).

## Features

- **Clear Scrollback** in the terminal tab context menu, plus a new **Ctrl+Shift+K** shortcut to clear the active pane from the keyboard ([#158](https://github.com/Eric-Ness/Tiki-V2/pull/164)). Shift is required so the shortcut doesn't collide with bash readline's `Ctrl+K` (`kill-line`). Fulfills ENHANCEMENT-IDEAS E1.
- **Resume Conversation banner** for terminals that previously ran `claude` ([#159](https://github.com/Eric-Ness/Tiki-V2/pull/167)). After a crash or restart, the banner offers one-click `claude --continue` so you don't have to retype it. Privacy: only commands matching `^claude` are persisted to localStorage — every other keystroke is dropped on the floor. New **Auto-resume Claude conversations** setting in Settings → Terminal opts into a 5-second auto-resume countdown.

## Bug Fixes

- **Ctrl+W and Ctrl+T no longer eat shell keys** ([#156](https://github.com/Eric-Ness/Tiki-V2/pull/163)). The window-level keydown handler used to intercept Ctrl+W (readline's `unix-word-rubout`) and Ctrl+T (`transpose-chars`) regardless of focus, killing the terminal mid-edit. Ctrl+W is now gated by an `.xterm`-focus check, Ctrl+T is no longer bound at all, and **Ctrl+Shift+T** opens new tabs.
- **PTY reader: UTF-8 carryover + 10ms IPC coalescing** ([#157](https://github.com/Eric-Ness/Tiki-V2/pull/165)). Two related defects, both fixed in one refactor: multi-byte UTF-8 sequences (emoji, CJK, decorated ANSI escapes) crossing the 4KB read boundary no longer turn into U+FFFD, and PowerShell tab-completion no longer fires hundreds of IPC events back-to-back. Decoded output now coalesces over a 10ms / 64KB window with a guaranteed final flush before `terminal-exit`. Supersedes ENHANCEMENT-IDEAS E4.

## Documentation

- **Settings explainer: Ctrl+Z in the terminal is SIGTSTP, not undo** ([#160](https://github.com/Eric-Ness/Tiki-V2/pull/166)). Static informational block in Settings → Terminal explaining that Ctrl+Z suspends the running program (resumable with `fg`), so users from editor backgrounds don't think their session crashed.

## Known limitations

- `lastCommand` detection in #159 is a keystroke-buffer heuristic: it misses Ctrl+R history recall, shell aliases (e.g. `tg` → `claude --continue`), `npx claude`, and arrow-key edits. The principled follow-up is OSC 133 shell integration — design carried forward so OSC 133 just becomes a different writer to the same `lastCommand` field.

## Deferred

- `@xterm/xterm` package migration ([#161](https://github.com/Eric-Ness/Tiki-V2/issues/161)) moved to v0.5.5. The package.json swap requires a `pnpm-lock.yaml` regeneration that this machine can't currently produce (Windows Defender blocks pnpm install workspace junctions — see CLAUDE.md). Will ship from a Linux host or after Defender exclusions land.
