# v0.6.5

Single-fix hotfix release: unblocks every fresh-install and post-auto-update v0.6.4 user currently sitting in a blank window.

## Fixes

- **Blank screen on fresh install / post-auto-update** ([#210](https://github.com/Eric-Ness/Tiki-V2/issues/210)) — Nine Zustand selectors across 8 components used inline `?? []` / `?? {}` fallbacks. When the project key was missing from the store (the fresh-install / no-active-project state), each call returned a freshly-allocated reference, so React 19's `useSyncExternalStore` saw a "new" snapshot every check and re-rendered until tripping the max-update guard. `<KanbanBoard>` mounted first and threw; with no error boundary above it in `App.tsx`, React unmounted the entire tree → blank WebView in production (no DevTools surface to show the error). Lifted `EMPTY_TABS` and `EMPTY_COLUMN_ORDER` to module-level constants exported from `terminalStore.ts` and `kanbanStore.ts`; all 9 callsites updated to use them. Stable references → no loop.

## Notes

- Latent since v0.5.x; masked for users with a non-null `activeProjectId` persisted from prior versions. The v0.6.3 → v0.6.4 auto-update reset that state for many users, surfacing the bug as a v0.6.4 regression.
- Bundle delta: +2 bytes (1,554,470 → 1,554,492). CSS hash unchanged. No new dependencies.
- Test coverage: shared 65 / desktop 122 / Rust 28 unchanged (no new behavior to test; the fix is a stability constant).
- **Follow-up worth filing**: add an error boundary above the panel sections in `App.tsx` so a future crash in one panel does not unmount the whole tree silently.
