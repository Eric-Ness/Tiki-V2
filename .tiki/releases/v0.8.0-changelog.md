# v0.8.0

The dependency graph gets a lot more informative. Nodes now tell you how far along each issue is, and clicking one shows exactly what "done" means for it — both updating live as work progresses.

## Features

- **Durable phase progress (N/M) on every node** (#256) — each dependency-graph node now shows which phase its issue is on (e.g. `2/4`), derived from the issue's plan. The indicator is durable for the whole graph: `0/N` for planned-but-not-started, `k/N` mid-execution, `N/N` for completed/shipped — not just for the one issue currently running. It updates live as phases complete.

- **Click a node to open a live success-criteria panel** (#257) — clicking a node opens a side panel listing that issue's success criteria as a checklist that ticks off (☐ → ☑, with an `N/M verified` count) as the phases covering each criterion complete. Nodes without a plan show a graceful empty state. The panel closes cleanly and is dismissed when you switch releases.

### Under the hood

- Phase progress is derived from the **durable** on-disk plan (`plan.phases[].status`) rather than the ephemeral live `activeWork.phase` (which is cleared on completion), so the indicator survives shipping.
- Wired up the previously-dead `planChanged` file-watch event: plan-file writes now bump a per-issue nonce that re-reads only the changed plan (no GitHub re-fetch), which is what makes both the N/M counter and the criteria panel tick live.
- The success-criteria checklist markup is now shared between the issue detail panel and the new graph panel via an extracted presentational component — no behavior change to the detail view.
