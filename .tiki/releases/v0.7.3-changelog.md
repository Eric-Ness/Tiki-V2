# v0.7.3 — version-bump self-syncs `.framework-version`

A small release-tooling fix, dogfooded by its own release: `pnpm version-bump 0.7.3` was the first invocation to bump `.tiki/.framework-version` automatically.

## Changed
- **`version-bump.mjs` now also bumps `.tiki/.framework-version`** ([#229](https://github.com/Eric-Ness/Tiki-V2/issues/229)) — The script bumped 4 version files (`package.json`, `tauri.conf.json`, `Cargo.toml`, `plugin.json`) but skipped `.tiki/.framework-version`, the 5th version file the release process tracks. It went stale after every `pnpm version-bump` and had to be fixed by hand on every release from v0.6.0 through v0.7.2. It's now a 5th entry in the script's `files[]` array; because the file is plain text (just the version string, no trailing newline), its `update()` rewrites it wholesale and preserves the format so there's no spurious diff. v0.7.3's own version bump is the live acceptance test — all 5 files moved to 0.7.3 in one invocation, no manual step.
