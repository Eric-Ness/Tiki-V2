# v0.8.2

## Bug Fixes
- **Shipped releases now correctly show as "completed" in the sidebar and detail panel** (#259) — the location-derived `archived` flag on a release was annotated `#[serde(skip_serializing)]`, which kept it off disk but also stripped it from the Tauri IPC payload sent to the frontend. The UI fell back to the on-disk `status`, which the ship teardown leaves stale at `"active"`, so shipped releases (e.g. v0.7.7, v0.8.1) rendered an "active" badge instead of "completed". This is the true root cause behind the earlier #255/#258 fixes, which addressed adjacent symptoms but never the serialization. The flag is now serialized (its persisted value is inert — it is re-derived from file location on every load); the sidebar and detail panel share one tested `isReleaseCompleted` helper so they can't drift; and a Rust guard test asserts the flag survives serialization across the IPC boundary.

## Internal
- **Releases now run a full test gate before publishing** — the release workflow runs typecheck, build, all test suites (shared / desktop / framework / Rust), and clippy before building or publishing any installer (the `release` job now depends on a `test` job). Previously the deploy path ran no tests at all, and direct-to-main releases bypassed PR CI entirely — which is how the earlier incomplete fix shipped unverified.
