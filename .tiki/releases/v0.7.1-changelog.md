# v0.7.1 — Write-integrity hardening

Wave 3 (part 1) of the status-desync epic ([#218](https://github.com/Eric-Ness/Tiki-V2/issues/218)) — internal reliability hardening, no UI changes. The remaining Wave 3 item (#223, collapsing the dual `activeWork` store) needs a live verification gate and ships separately in v0.8.0.

## Bug Fixes
- **Write integrity — `state.mjs` lock + watcher active-project repoint** ([#224](https://github.com/Eric-Ness/Tiki-V2/issues/224)) — Two fixes. **(1)** `state.mjs` serialized its `transition`/`remove`/`append-history` read-modify-write behind an exclusive lockfile (stale-steal + timeout + exit-cleanup, Node built-ins only), so writes chained during a release cascade can no longer interleave and lose an update (the transient `exit 2` seen during release teardowns). **(2)** The desktop file watcher anchored on `cwd/.tiki` with a doubled-`.tiki` path bug and only repointed on a manual project switch; it now resolves the project root correctly and self-terminates a superseded watcher, so after startup-restore it observes the active project's `.tiki` instead of a stale one.
