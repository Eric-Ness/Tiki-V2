# v0.9.1

Desktop performance pass — faster cold start and lower terminal latency.

## Performance

- **Faster cold start** (#263) — the production bundle is now code-split: a `manualChunks` vendor split plus `React.lazy` + first-visit gating for the Kanban, Dependencies, and Settings views and the detail panels. The single ~1.5 MB JS chunk drops to a ~192 kB entry, and the graph/dnd/markdown vendor weight (~649 kB) now loads only when those views are first opened — roughly a 47% reduction in cold-start JS. The terminal view stays eager so PTYs are never torn down.
- **Lower terminal latency** (#264) — a single process-wide `terminal-output`/`terminal-exit` listener now dispatches to terminals by id (replacing the per-terminal listener fan-out, so one PTY event wakes one callback instead of all open terminals), and the PTY reader's coalesce/idle timings were tightened (4 ms flush window + an adaptive idle nap) to cut the keystroke-echo floor from ~10–20 ms to ~4–6 ms without busy-spinning.
