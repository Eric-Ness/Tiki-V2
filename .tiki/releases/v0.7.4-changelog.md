# v0.7.4 — refactor backlog: structural cleanup & drift guards

A maintainability-focused release. A structural deep-dive surfaced a backlog of god-files, duplication, and missing tests; this release lands all of it — plus one confirmed type-drift bug fix — with no user-facing behavior changes.

## Fixed
- **PhaseStatus type drift** ([#231](https://github.com/Eric-Ness/Tiki-V2/issues/231)) — the desktop hand-redeclared `PhaseStatus` and had dropped the canonical `skipped` (while carrying a non-canonical `running`), so a genuinely skipped phase was untyped on the frontend. The desktop now sources `WorkStatus`/`PhaseStatus`/`PipelineStep` from `@tiki/shared`, and the shared package now exports `PipelineStep`/`PipelineStepRecord` (previously defined but never surfaced through its public API — the root cause that forced the desktop to hand-roll them).

## Internal / Refactoring
- **De-duplicated the workflow command files** ([#230](https://github.com/Eric-Ness/Tiki-V2/issues/230)) — the `/tiki:*` command `.md` files existed in three trees that had drifted (the dogfood `/tiki:release` had even regressed to the pre-parallel-dispatch version). Removed the orphaned tree, made the dogfood copy a generated artifact, and added a CI parity guard. The framework `node:test` suite now runs in CI for the first time.
- **Split `github.rs`** ([#233](https://github.com/Eric-Ness/Tiki-V2/issues/233)) — the 1107-line backend file (and the only complex Rust module with zero tests) is now a cohesive `github/` module (issues / pulls / releases / labels / git / auth / rate-limit) with 13 new `gh`-output parse tests.
- **Decomposed `App.tsx`** ([#234](https://github.com/Eric-Ness/Tiki-V2/issues/234)) — the 722-line shell (722 → 460) split into `useTikiFileSync` / `useGithubFreshness` / `useBulkYoloCascade` hooks; the bulk-YOLO cascade is now a pure, unit-tested function (previously untestable inside the watcher callback). The fresh-ref crash guard now covers the new hooks.
- **Extracted `config.rs`** ([#235](https://github.com/Eric-Ness/Tiki-V2/issues/235)) from the 965-line `commands.rs`.
- **Cross-language transition parity tests** ([#232](https://github.com/Eric-Ness/Tiki-V2/issues/232)) — shared fixtures now guard the state-transition mutation body (phase-clear / parallelExecution-clear / parentRelease-preserve) across the Node shim and the Rust mirror, beyond the already-guarded transition table.
- **Derived the desktop view-model types from `@tiki/shared`** ([#237](https://github.com/Eric-Ness/Tiki-V2/issues/237)) — `IssueContext` / `ReleaseContext` and `activeWork` now derive from the shared work types via `Pick`/`Omit`, so a rename or shape change in shared becomes a compile error instead of silent drift.
