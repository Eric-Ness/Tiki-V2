# v0.6.3

A focused follow-up release: completes 3 of the 4 batch-operations items deferred from v0.6.2 #96. Bulk YOLO is the only remaining item — re-deferred to [#206](https://github.com/Eric-Ness/Tiki-V2/issues/206) for a proper UX/state design pass.

## Features

- **Batch operations follow-up: shift+click range, bulk remove-label, bulk assign-to-release** ([#203](https://github.com/Eric-Ness/Tiki-V2/issues/203)) — Three additions to v0.6.2's batch-ops infrastructure:
  - **Shift+click range selection.** Per-surface anchor tracking in `useSelectionStore` (kanban columns and the issues list each get their own anchor). Plain checkbox click sets the anchor; shift+click selects everything between anchor and target in the surface's visible-issue order. Falls back to plain toggle when the anchor is no longer visible.
  - **Bulk remove-label.** New action in `BulkActionToolbar` symmetric with bulk Add Label. The label picker shows the **union of labels currently applied to the selection** (not all repo labels) — fetches each selected issue via `fetch_github_issue_by_number` and dedupes by name. Empty-union case shows a clear message.
  - **Bulk assign-to-release.** Opens the existing `ReleaseDialog` pre-populated with the selected issues. Took the `openDialog(release?, opts?)` path with a new `initialIssues` opts arg rather than synthesizing a fake `version: ''` release object — the dialog's `editingRelease` triggers an "Edit Release" title that would have shown an empty version, wrong UX.
