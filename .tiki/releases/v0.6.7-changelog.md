# v0.6.7

Single-issue patch — fixes a release-pipeline data-loss bug (Wave 1 / MVP of epic #218).

## Bug Fixes

- **Release-shipped issues now reach the Kanban Completed column** (#219). `/tiki:release` children were removed from `activeWork` but never added to `history.recentIssues`, and the Completed column read **only** `recentIssues` — so release-shipped issues vanished from the board entirely. Fixes:
  - The Completed column now unions `recentIssues` + `recentReleases[].issues` (deduped by number), and the cross-column exclusion set is uncapped — **retroactively surfacing past releases' children (including v0.6.6's #214/#215/#216) with no `state.json` migration**.
  - `release.md` teardown now appends per-child issue history (mirrors `ship.md`), guarded by a coverage-test assertion.
  - `state.mjs append-history` is now idempotent (dedupe by issue number / release version), so the per-child append is safe even when `ship.md` already appended during the cascade.
