# v0.6.0 — "UX polish"

The first backlog-driven release since v0.5.0 — a coherent terminal-and-detail-panel polish pass that breaks the v0.5.4–v0.5.7 single-issue reactive-bugfix streak. Five enhancements from `docs/ENHANCEMENT-IDEAS.md` (E2, E3, E10, E11, E14), scoped and shipped together.

## Features

- **Regex toggle in terminal search** (#173) — a `.*` toggle beside the `Aa` case toggle in the terminal search overlay. `@xterm/addon-search` already supports `regex`; the UI now exposes it and forwards it through every search call site. Uncompilable patterns are caught before reaching the addon — the input shows a red border and the count reads "Invalid regex."
- **Runtime terminal font-size** (#174) — `Ctrl+=` / `Ctrl++` increase, `Ctrl+-` decreases, and `Ctrl+0` resets the live terminal's font size, reflowing the grid immediately. Runtime-only, consistent with the "settings apply to new terminals only" model; `Ctrl+Shift+-` (readline undo) still passes through to the shell.
- **Jump-to-terminal from the detail panel** (#175) — issue-scoped commands dispatched from the kanban, the issues sidebar, or the command palette now record which terminal they ran in. The issue detail panel shows a "Jump to terminal" button that switches to the Terminal view, the right tab, and focuses the pane. The button is hidden when there's no live associated terminal.
- **Illustrated detail-panel empty state** (#176) — the bare "Select an issue" fallback is replaced with a GET → REVIEW → PLAN → AUDIT → EXECUTE → SHIP pipeline diagram and a `Ctrl+K` command-palette hint, giving new users workflow surface area on first open.
- **Recovery dialog surfaces the parse-failure line** (#177) — the state-recovery dialog now shows a structured "Parse error at line N, column M" callout, and renders backup previews as line-numbered rows. For invalid backups, the offending line is highlighted and scrolled into view.

## Internal

- New unit tests: `TerminalSearch.test.ts`, `terminalStore.test.ts`, and `parseJsonErrorLocation` coverage in `recoveryFlow.test.ts` — desktop vitest suite grew from 76 to 89 cases.
