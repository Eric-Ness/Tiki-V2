# v0.8.1

## Bug Fixes
- **Shipped releases no longer show a stale `active` badge** (#258) — the file-watcher's release reload (`useTikiFileSync`) called `load_tiki_releases` without `includeArchived`, then fully replaced the release list. On any `releaseChanged` event (which fires during a ship as the release file is created and then moved to `archive/`), shipped releases were dropped from the sidebar or frozen at a stale `active` badge until a manual refresh. The watcher reload now mirrors the sidebar mount loader by passing `includeArchived: true`. This completes the #255 archived-aware loading fix (the mount path was fixed there; this watcher path was missed). Added a source-scan regression guard so every `load_tiki_releases` call site must pass the flag, and corrected a stale doc comment in `commands.rs`.
