# v0.6.4

A focused single-feature release: closes the **last** v0.6.2 #96 deferral. Bulk YOLO now ships — the desktop can enqueue a multi-selection of issues and run `/tiki:yolo` on each in turn, with progress visible in a non-blocking dialog.

## Features

- **Bulk YOLO cascade** ([#206](https://github.com/Eric-Ness/Tiki-V2/issues/206)) — Three pieces:
  - **Persisted queue store** (`useBulkYoloStore`): per-project, survives desktop restarts via `persist` middleware so a crash mid-cascade can be resumed. Tracks queue snapshot, current index, completed list, failures, status (`idle | running | paused | failed`), and the dispatch terminal id. Actions: `startRun`, `advance`, `recordFailure`, `pause`, `resume`, `abort`.
  - **Auto-advance controller** wired into the existing `tiki-file-changed` listener (added in v0.6.2 #196). When the queue's current issue moves from `activeWork` into `history.recentIssues`, the controller calls `advance()` and dispatches the next `/tiki:yolo {N}` to the run's terminal via the existing `write_terminal` IPC. When the current issue's state flips to `failed`, the controller calls `recordFailure(reason)` and surfaces an error toast — pause-on-failure semantics matching `/tiki:release`.
  - **`<BulkYoloDialog>`** floating bottom-right card (NOT modal-blocking — the user keeps using the app). Shows queue with status icons (✓ done, ▶ current, ○ pending, ✗ failed), current-issue progress, failures collapse, target terminal name, Pause/Resume/Abort buttons. Mounted alongside `<BulkActionToolbar>`. New 'Run YOLO' button in the toolbar starts the cascade — uses the *active* terminal as the dispatch target (no auto-create; surfaces a clear toast if no terminal is open).

## Notes

- Test coverage: desktop vitest 114 → 122 (+5 store + 3 dispatch helper). Shared 65 / Rust 28 unchanged.
- No new dependencies in either Rust or JavaScript.
- Pre-tag verification gate honored: all 5 version files bumped (`tauri.conf.json`, `Cargo.toml`, `plugin.json`, `package.json`, `.tiki/.framework-version`); `pnpm build` + `pnpm test` green; release-PR CI watched green before merge.
- This release closes the *last* deferred item from v0.6.2 #96. v0.6.2 → v0.6.3 → v0.6.4 chained the original batch-ops MVP into a fully-realized feature set across three releases.
- **Design decisions baked into this release** (from the open questions in #206's body):
  - **Single shared terminal** for the cascade (not N terminals — keeps it observable in one place).
  - **Pause-on-hard-failure**, surface what failed, user resumes manually after fixing — same pattern as `/tiki:release`.
  - **Progress UI = floating dialog**, not modal — user shouldn't be blocked from doing other work while a long cascade runs.
  - **Resume across desktop restarts** via persist middleware (the queue snapshot survives).
  - **Standalone primitive** for now; sharing with `/tiki:release`'s wave dispatcher is left for a future unification issue.
