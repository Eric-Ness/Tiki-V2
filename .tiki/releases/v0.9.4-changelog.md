# v0.9.4

Epic 3 "Guards that test behavior" — the guards and abstractions that stop the recurring bug *meta-patterns* the issue archaeology surfaced, plus a lint cleanup. Internal hardening; no behavior changes beyond one kanban UX improvement.

## Features / UX
- **Kanban gives feedback on every drag** (#280) — backward/lateral drags that the board can't dispatch now show a toast ("drive it from the terminal") instead of silently snapping back. The column move table is parity-tested against the canonical transition table so the board can never offer a move the state machine forbids.
- **Pending visual verifications are now a checklist** (#281) — Settings → Diagnostics surfaces success criteria from shipped plans that were left unverified (visual/manual ones that can't be auto-checked), and the release-readiness gate soft-warns about them. No more relying on prose hand-off.

## Reliability / hardening
- **Behavior-level guard tests** (#278) — every struct crossing the Tauri IPC boundary is round-tripped through serde to prove its fields survive on the wire (generalizes the #259 fix), and a source-scan bans raw `Command::new` outside the `CREATE_NO_WINDOW` wrapper (the Windows console-popup class).
- **The three declined abstractions, built** (#279) — one shared `compareSemver`, one `resolve_tiki_path`, and an ESLint rule catching the inline `?? {}`/`?? []` fresh-reference pattern (the blank-screen crash class) in Zustand selectors and hook deps.
- **Lint is clean and gated** (#282) — `pnpm lint` errors cleared via a deliberate config (ignore build artifacts, honor `_`-prefixed discards, warn-level for React-Compiler advisory rules the project hasn't adopted) and now runs in CI so it can't rot again.
