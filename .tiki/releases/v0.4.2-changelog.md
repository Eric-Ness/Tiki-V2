# v0.4.2

Six visual refinements from the `docs/ENHANCEMENT-IDEAS.md` backlog, themed around making the desktop feel more polished and alive.

## Visual polish

- **Semantic `WorkStatus` color tokens** (E49) ŌĆö `--status-pending / --status-reviewing / --status-planning / --status-executing / --status-paused / --status-shipping / --status-completed / --status-failed` now live in `index.css` for both dark and light themes. Hardcoded hex values in `WorkProgressCard.css` have been replaced with `var(--status-*)` references. Light theme uses darker, less-saturated variants for legibility on white. Previously-missing status badge styles for `reviewing` and `shipping` were added.

- **Running phase segment pulses in executing-green** (E50) ŌĆö the animation existed but used paused-blue and cycled hue between two shades. Now anchored to `--status-executing` with opacity-only modulation plus a soft green glow. Reads as "this phase is active" rather than "this status is changing."

- **Pulse on busy terminal-tab dots** (E56) ŌĆö `StatusDot` in the tab strip now scales and glows when `status === 'busy'`. Other states (idle / ready / starting / exited) stay static so the visual signal is precise.

- **Accent ring on card hover** (E54) ŌĆö `WorkProgressCard.clickable:hover` and `IssueCard:hover` add a 1px accent-color ring via `box-shadow`. Uses shadow (not `border-color` shorthand) so the status-meaningful `border-left` is preserved. Replaces the previous sub-threshold `rgba(255,255,255,0.05)` background shift with a clearer interactive affordance.

- **Selected-card elevation** (E53) ŌĆö `IssueCard.selected` and `KanbanCard--selected` gain a 2px accent ring + 4px soft drop-shadow + `z-index` lift. Hierarchy now reads on two axes (color + elevation) instead of color alone.

## Accessibility

- **Respect `prefers-reduced-motion`** (E52) ŌĆö two layers: (a) a global CSS rule collapses every CSS animation and transition to 0.01ms under `@media (prefers-reduced-motion: reduce)`; (b) `<MotionConfig reducedMotion="user">` wraps the app so every framer-motion consumer (Kanban `AnimatePresence`, column count badge, Pipeline Timeline) respects the OS preference. Implemented while animation surface is still small ŌĆö the cost grows the more `motion.*` consumers land.

## Notes

- Five of six items are pure CSS edits. The only TypeScript edit is the `<MotionConfig>` wrapper in `App.tsx` and a one-line modifier-class addition in `TerminalTabs.tsx`'s `StatusDot`.
- The previous v0.4.1 quick-wins bundle already referenced `var(--accent-color)`; v0.4.2's tokens follow the same pattern and adopt the convention for status colors specifically.
- Sequenced so foundational tokens (E49) ship before consumers (E50, E54, E62-future), and animations (E50, E56) ship before the reduced-motion gate (E52).
