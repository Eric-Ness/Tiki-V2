# v0.9.2

Dependency-graph UX papercuts — auto-fit, smoother hover, and success criteria that survive completion.

## Improvements

- **Graph auto-fits to view** (#266) — the dependency graph now frames itself automatically on first load and when you switch releases, instead of opening off-screen until you click "Fit View". The auto-fit is keyed on the set of nodes, so it won't yank the viewport out from under you during live status updates or while you're panning. The manual Fit View button is still there.
- **No more node hover flicker** (#266) — node components are now memoized, so hovering to trace a dependency lineage no longer re-renders every other node's contents. The blink on hover is gone; the lineage highlight still works.

## Bug Fixes

- **Success criteria no longer vanish after an issue ships** (#266) — clicking a completed issue's node in the dependency graph kept showing an empty panel because the plan lookup only checked `.tiki/plans/` and a shipped issue's plan has moved to `.tiki/plans/archive/`. The lookup now falls back to the archive, so the success-criteria checklist renders (all-verified) for completed issues — the same "look in archive too" pattern used for completed releases (#255/#258).
