# v0.2.17

## Features

- **Research command and knowledge capture** ([#102](https://github.com/Eric-Ness/Tiki-V2/issues/102)) — New `/tiki:research <topic>` command writes findings to `.tiki/research/<topic>.md` with YAML front-matter. The REVIEW and PLAN steps now auto-capture domain learning as a side-effect; PLAN and EXECUTE auto-retrieve relevant docs and pass them to sub-agents. Desktop sidebar gains a Research section that lists docs and renders them with the existing markdown renderer.

- **Parallel phase execution** ([#99](https://github.com/Eric-Ness/Tiki-V2/issues/99)) — Independent phases (no dependencies, no overlapping `files`) now run concurrently via single-message multi-Task sub-agent dispatch. EXECUTE uses Kahn's-by-levels topo grouping with greedy file-conflict splitting. State tracks parallel groups via a new optional `parallelExecution` field; the desktop sidebar renders a "(parallel: N)" badge when more than one phase runs concurrently.

- **Automated test framework integration** ([#100](https://github.com/Eric-Ness/Tiki-V2/issues/100)) — EXECUTE and SHIP now detect and run the project's test framework (vitest, jest, pytest, cargo test, go test). Test results parse into a `testResults` field surfaced in phase summaries. `/tiki:ship` runs the full test suite before committing; failures present a skip/pause/abort menu. Configurable via `.tiki/config.json` `workflow.tests`.

- **Auto-healing for failed phases** ([#101](https://github.com/Eric-Ness/Tiki-V2/issues/101)) — Opt-in heal loop: when phase verification fails and `workflow.autoHeal.enabled` is true, EXECUTE categorizes the error (build, type, test, lint), applies a targeted fix, re-runs verification, and retries up to `maxAttempts` before falling back to manual recovery. All heal attempts persist as `HealAttempt` records in phase progress for transparency.
