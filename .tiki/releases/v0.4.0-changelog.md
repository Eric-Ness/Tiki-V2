# v0.4.0

Two follow-up improvements on v0.3.0's reliability foundation.

## Features

- **State machine transition validation** ([#103](https://github.com/Eric-Ness/Tiki-V2/issues/103)) — New canonical `WorkStatus` transition table in `@tiki/shared` with `canTransition` and `assertTransition` helpers. The Rust `state_transition` IPC and the JS shim's `LEGAL` constant now both mirror the canonical table. Removes the previously-allowed `failed → shipping` transition (which the original spec called out as invalid) and adds the missing `executing → completed` short-circuit path. Closes the only known framework enforcement gap by routing `review.md`'s `pending → reviewing` transition through the validated shim instead of a direct JSON write. Adds 47 vitest cases in `@tiki/shared`.

- **Terminal search** ([#87](https://github.com/Eric-Ness/Tiki-V2/issues/87)) — `@xterm/addon-search` wired into each `Terminal` leaf with a `TerminalSearch` overlay component. `Ctrl+F` opens the bar (scoped to the active split pane), `Escape` closes and restores focus to xterm, `Enter`/`Shift+Enter` and `F3`/`Shift+F3` navigate matches. Search-as-you-type with 150 ms debounce, case-insensitive default with an `Aa` toggle, no-match state with red border. Adds 8 vitest cases for the pure helpers.

## Deferred to next release

- **Project templates for `.tiki/` initialization** ([#114](https://github.com/Eric-Ness/Tiki-V2/issues/114)) — Originally scoped to v0.4.0, deferred after a local Windows env issue blocked verification mid-session. Returns to the open backlog.

## Other

- Adopted v0.3.0 framework commands into the project's own `.claude/commands/tiki/*` copies, replacing direct-JSON state writes with `state.mjs` shim calls.
- Gitignored local runtime artifacts (`.claude/worktrees/`, `.tiki/backups/`, `.tiki/issues/`).
- Captured review notes for issues #116 and #118 under `.tiki/research/`.
