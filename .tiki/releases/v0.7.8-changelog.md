# v0.7.8

## Features
- **Completed releases are no longer detail "dead ends"** (#255) — clicking a shipped release used to open a near-empty view, because shipping archives the release's def file off-disk and the sidebar then routed the click to the bare GitHub release panel. Completed releases now open the rich Tiki release detail with a **Timeline** (created / completed / duration), the **included issues**, the **release notes / changelog body**, and a **"View on GitHub"** link — while the workflow actions (Edit / Run Release / Review / Delete) are hidden on shipped history. Loading archived releases also closes the brief window where a just-shipped release could vanish from the sidebar before CI published its GitHub release.

### Under the hood
- `archived` is now derived from a release file's location (`releases/archive/`) rather than its `status` field (which the ship teardown leaves stale), and is never persisted.
- New backend commands: `read_release_changelog` (local `{version}-changelog.md`, archive fallback) and `fetch_github_release_url` (`gh release view --json url`, degrades gracefully to no link during the archive→CI-publish gap).

Scope note: closed-issue field enrichment, per-release status-history persistence, and GitHub-API caching of immutable records were intentionally deferred as future work.
