# v0.3.0

Foundational reliability release: a structural fix for fragile state contracts, an automated test suite + PR CI gate, and user-facing corruption recovery.

## Features

- **Typed `state_transition` Tauri IPC and Node CLI shim for state.json mutations (#144)** — Replaces fragile prose-based "you MUST write this JSON" contracts in framework commands with a typed Rust IPC backed by an explicit state-machine. The new `state_transition` command validates legal status/step transitions, preserves `parentRelease` automatically, and bumps `lastActivity`. A zero-dependency Node CLI shim (`packages/framework/scripts/state.mjs`) gives framework commands a bash-reachable entry point to the same logic. Framework prose for state management shrank by 60% (17,366 → 6,948 bytes) while keeping the legacy direct-JSON-write path supported for backward compatibility. 10 Rust unit tests cover legal/illegal transitions, paused/failed recovery, `parentRelease` preservation, fresh-create, and `lastActivity` bumping.

- **Automated test suite — Rust fixtures, Vitest store tests, and PR CI workflow (#145)** — The repo went from zero tests to **58 passing tests** in one release. New Rust integration test (`tests/state_format_compat.rs`) loads 6 state-file fixtures (legacy-flat, legacy-phases-object, legacy-phases-array, canonical-current, with-parallel-execution, with-parent-release) and round-trips each through `TikiState` to pin the existing legacy serde shims. Vitest is now configured for `apps/desktop/` with 41 tests across `splitTree`, `commandPaletteStore`, `tikiStateStore`, and `executeCommand` — enabled by extracting pure helpers (`splitTree.ts`, `executeCommand.ts`) from previously component-coupled logic. New `.github/workflows/pr.yml` runs `pnpm typecheck && pnpm build`, `vitest`, `cargo test`, and `cargo clippy --deny warnings` on every pull request. `pnpm test` now works from repo root.

- **Corruption recovery UX — one-click backup restore for unparseable state.json (#146)** — When `.tiki/state.json` becomes corrupt or unparseable (after `read_json_resilient` retries are exhausted), a new `StateRecoveryDialog` opens with a list of available backups sorted newest-first. Each backup shows timestamp, size, and approximate age; backups that themselves don't parse are grayed out and disabled. Restore wraps the existing `restore_backup` primitive with a safety step: the current corrupt state is snapshotted to `.tiki/backups/state.{ts}.broken.json` (excluded from retention pruning) before the restore overwrites. The dialog also offers "Edit Manually" (opens the file in the system editor via `tauri-plugin-shell`) and "Start Fresh" (requires literally typing `reset` in a confirmation input). 24 new Vitest tests for the pure recovery-flow helpers, 10 new Rust unit tests for the safety-backup logic.
